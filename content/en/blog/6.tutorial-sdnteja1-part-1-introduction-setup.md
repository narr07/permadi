Welcome to the modern school website development tutorial series! In this series, we will dissect the process of building a website for **SDN Teja 1** using the latest technologies in today's web development ecosystem.

Building a school website is not just about displaying static information, but also about how to provide a fast user experience, attractive design, and ease of content management for school admins.

## Tech Stack Used

In the **SDN Teja 1** project, we use a combination of very modern technologies to ensure maximum performance and ease of development:

1.  **Next.js 16 (App Router):** The leading React framework for server-side rendering (SSR) and performance optimization.
2.  **Sanity CMS v5:** A highly flexible headless Content Management System (CMS) for managing articles, news, and teacher data.
3.  **Tailwind CSS v4:** The latest version of the utility-first CSS framework for super-fast styling.
4.  **React 19:** Utilizing the latest React features for smoother interactions.
5.  **Shadcn UI & Radix UI:** Accessible and easily customizable UI components.

## Step 1: Initialize Next.js Project

The first step is to create a new Next.js project. You can use the following command in your terminal:

```bash
npx create-next-app@latest sdnteja1 --typescript --tailwind --eslint
```

Make sure you choose the **App Router** option as we will be utilizing modern features like Server Components.

## Step 2: Sanity CMS Integration

Once the Next.js project is ready, the next step is to add Sanity to the project. Sanity will be the "brain" behind our website's content.

```bash
npx sanity@latest init
```

This command will guide you through the process of creating a dataset and project ID in Sanity. Once finished, you will have a `sanity/` directory in your project root.

## Step 3: Directory Structure

The SDN Teja 1 website has a clean directory structure to separate page logic, components, and data management:

- `app/`: Contains routes for each page (Articles, News, Profile, etc.).
- `components/`: UI components such as Hero, Navbar, and Card.
- `sanity/`: Contains schemas for the data types we need.
- `lib/`: Contains utility functions and Sanity client configuration.
