In this second part, we will focus on the "brain" of our website, which is **Sanity CMS**. Sanity allows us to define highly flexible data structures without the hassle of managing databases manually.

## What is a Sanity Schema?

In Sanity, a **schema** is how we define the data types we will manage. For the **SDN Teja 1** website, we need several document types, such as:

1. **Articles:** For literacy content and free writing.
2. **News:** For official school announcements.
3. **Teachers:** For teaching staff profiles.
4. **Activities:** For documenting school activities.

## Creating an Article Schema

Let's break down how to create a schema for **Articles**. We use the `defineType` and `defineField` functions from Sanity to define the document structure.

Example file `sanity/schemaTypes/artikelType.ts`:

```typescript
import {DocumentTextIcon} from '@sanity/icons'
import {defineField, defineType} from 'sanity'

export const artikelType = defineType({
  name: 'artikel',
  title: 'Artikel',
  type: 'document',
  icon: DocumentTextIcon,
  fields: [
    defineField({
      name: 'judul',
      type: 'string',
    }),
    defineField({
      name: 'slug',
      type: 'slug',
      options: {
        source: 'judul',
      },
    }),
    defineField({
      name: 'guru',
      type: 'reference',
      to: {type: 'guru'}, // Reference to the teacher document
    }),
    defineField({
      name: 'gambar',
      type: 'image',
      options: { hotspot: true },
    }),
    defineField({
      name: 'publishedAt',
      type: 'datetime',
    }),
    defineField({
      name: 'isi',
      type: 'blockContent', // Rich text content type (Portable Text)
    }),
  ],
})
```

## Adding Schema to the Index

After defining the schema, we must register it in the `sanity/schemaTypes/index.ts` file so that Sanity Studio can display it:

```typescript
import {artikelType} from './artikelType'
import {guruType} from './guruType'
import {beritaType} from './beritaType'
// ... import other schemas

export const schema = {
  types: [artikelType, guruType, beritaType, /* ... */],
}
```

## Why Use Sanity?

Sanity provides **Real-time Collaboration** and **Visual Editing** features that are very helpful. School admins can see a live preview of the content as they write an article. Additionally, with **GROQ** queries, we can fetch data from Sanity very efficiently according to our Next.js website's needs.

## Conclusion Part 2

We now have a solid data structure in Sanity CMS. In the next tutorial (Part 3), we will start diving into the world of **Next.js** to build the interface (UI) and display data from Sanity on the website pages.

We will learn how to create the **Navbar**, **Hero** components, and how to perform GROQ queries to fetch the list of latest articles. See you in the next part!
