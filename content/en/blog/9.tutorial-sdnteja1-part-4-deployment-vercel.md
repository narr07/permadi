Congratulations! You've reached the end of the **SDN Teja 1** website development tutorial series. We've gone through the stages of initialization, Sanity CMS integration, and developing a cool interface with neo-brutalist style. Now, it's time to publish our work to the real world.

## Preparation Before Deployment

Before sending code to Vercel, there are a few things we must ensure:

1. **GitHub Repository:** Make sure your project has been uploaded to GitHub (or GitLab/Bitbucket).
2. **Environment Variables:** Our website depends on Sanity. We need `NEXT_PUBLIC_SANITY_PROJECT_ID` and `NEXT_PUBLIC_SANITY_DATASET`.
3. **Sanity CORS:** Don't forget to add your production URL (e.g., `https://sdnteja1.vercel.app`) to the CORS origin list in the Sanity dashboard so the API can be accessed from that domain.

## Step 1: Connect to Vercel

Vercel is the most popular platform for deploying Next.js applications due to its very seamless integration:

1. Log in to the **Vercel** dashboard.
2. Click the **"Add New"** button then select **"Project"**.
3. Connect your GitHub account and select the `sdnteja1` repository.

## Step 2: Set Environment Variables

On the Vercel project configuration page:

- Click the **Environment Variables** tab.
- Add `NEXT_PUBLIC_SANITY_PROJECT_ID` with the value from your Sanity project.
- Add `NEXT_PUBLIC_SANITY_DATASET` (usually `production`).
- Add an API token if you need to access private data.

## Step 3: Deploy!

Click the **"Deploy"** button. Vercel will automatically run the build process:

- Installing dependencies.
- Running `next build`.
- Optimizing image assets and scripts.

Wait a few minutes, and your website will soon be online with a `.vercel.app` domain.

## Sanity Webhook Integration (Optional but Important)

To ensure your website always displays the latest data whenever there's a change in Sanity Studio, you can use **Webhooks**. In the Sanity dashboard, add a webhook pointing to your Vercel deployment URL to perform automatic content revalidation.
