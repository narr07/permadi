Pada bagian kedua ini, kita akan fokus pada "otak" dari website kita, yaitu **Sanity CMS**. Sanity memungkinkan kita untuk mendefinisikan struktur data yang sangat fleksibel tanpa harus repot mengurus database secara manual.

## Apa Itu Skema Sanity?

Dalam Sanity, **skema** adalah cara kita mendefinisikan tipe data yang akan kita kelola. Untuk website **SDN Teja 1**, kita memerlukan beberapa tipe dokumen, seperti:

1. **Artikel:** Untuk konten literasi dan tulisan bebas.
2. **Berita:** Untuk pengumuman resmi sekolah.
3. **Guru:** Untuk profil tenaga pendidik.
4. **Kegiatan:** Untuk dokumentasi aktivitas sekolah.

## Membuat Skema Artikel

Mari kita bedah bagaimana cara membuat skema untuk **Artikel**. Kita menggunakan fungsi `defineType` dan `defineField` dari Sanity untuk mendefinisikan struktur dokumen.

Contoh file `sanity/schemaTypes/artikelType.ts`:

```typescript
import {DocumentTextIcon} from '@sanity/icons'
import {defineField, defineType} from 'sanity'

export const artikelType = defineType({
  name: 'artikel',
  title: 'Artikel',
  type: 'document',
  icon: DocumentTextIcon,
  fields: [
    defineField({
      name: 'judul',
      type: 'string',
    }),
    defineField({
      name: 'slug',
      type: 'slug',
      options: {
        source: 'judul',
      },
    }),
    defineField({
      name: 'guru',
      type: 'reference',
      to: {type: 'guru'}, // Referensi ke dokumen guru
    }),
    defineField({
      name: 'gambar',
      type: 'image',
      options: { hotspot: true },
    }),
    defineField({
      name: 'publishedAt',
      type: 'datetime',
    }),
    defineField({
      name: 'isi',
      type: 'blockContent', // Tipe konten kaya teks (Portable Text)
    }),
  ],
})
```

## Memasukkan Skema ke Index

Setelah mendefinisikan skema, kita harus mendaftarkannya di file `sanity/schemaTypes/index.ts` agar Sanity Studio bisa menampilkannya:

```typescript
import {artikelType} from './artikelType'
import {guruType} from './guruType'
import {beritaType} from './beritaType'
// ... import skema lainnya

export const schema = {
  types: [artikelType, guruType, beritaType, /* ... */],
}
```

## Mengapa Menggunakan Sanity?

Sanity memberikan fitur **Real-time Collaboration** dan **Visual Editing** yang sangat membantu. Admin sekolah bisa melihat pratinjau konten secara langsung saat mereka menulis artikel. Selain itu, dengan query **GROQ**, kita bisa mengambil data dari Sanity dengan sangat efisien sesuai kebutuhan website Next.js kita.
