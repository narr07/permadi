Selamat datang di seri tutorial pembangunan website sekolah modern! Pada seri ini, kita akan membedah proses pembuatan website untuk **SDN Teja 1** menggunakan teknologi terbaru di ekosistem pengembangan web saat ini.

Membangun website sekolah bukan hanya sekadar menampilkan informasi statis, tetapi juga tentang bagaimana memberikan pengalaman pengguna yang cepat, desain yang menarik, dan kemudahan dalam pengelolaan konten bagi admin sekolah.

## Tech Stack yang Digunakan

Dalam proyek **SDN Teja 1**, kita menggunakan kombinasi teknologi yang sangat modern untuk memastikan performa maksimal dan kemudahan pengembangan:

1.  **Next.js 16 (App Router):** Framework React terkemuka untuk rendering sisi server (SSR) dan optimasi performa.
2.  **Sanity CMS v5:** Content Management System (CMS) berbasis *headless* yang sangat fleksibel untuk mengelola artikel, berita, dan data guru.
3.  **Tailwind CSS v4:** Versi terbaru dari framework CSS utility-first untuk styling yang super cepat.
4.  **React 19:** Memanfaatkan fitur terbaru React untuk interaksi yang lebih mulus.
5.  **Shadcn UI & Radix UI:** Komponen antarmuka yang aksesibel dan mudah dikustomisasi.

## Langkah 1: Inisialisasi Proyek Next.js

Langkah pertama adalah membuat proyek Next.js baru. Anda bisa menggunakan perintah berikut di terminal:

```bash
npx create-next-app@latest sdnteja1 --typescript --tailwind --eslint
```

Pastikan Anda memilih opsi **App Router** karena kita akan memanfaatkan fitur-fitur modern seperti Server Components.

## Langkah 2: Integrasi Sanity CMS

Setelah proyek Next.js siap, langkah selanjutnya adalah menambahkan Sanity ke dalam proyek. Sanity akan menjadi "otak" di balik konten website kita.

```bash
npx sanity@latest init
```

Perintah ini akan memandu Anda melalui proses pembuatan dataset dan project ID di Sanity. Setelah selesai, Anda akan memiliki direktori `sanity/` di dalam root proyek Anda.

## Langkah 3: Struktur Direktori

Website SDN Teja 1 memiliki struktur direktori yang rapi untuk memisahkan logika halaman, komponen, dan pengelolaan data:

- `app/`: Berisi route untuk setiap halaman (Artikel, Berita, Profil, dll).
- `components/`: Komponen UI seperti Hero, Navbar, dan Card.
- `sanity/`: Berisi skema (schema) untuk tipe data yang kita butuhkan.
- `lib/`: Berisi fungsi utilitas dan konfigurasi client Sanity.
