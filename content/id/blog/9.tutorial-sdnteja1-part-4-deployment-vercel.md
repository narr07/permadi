Selamat! Anda telah sampai di bagian akhir dari seri tutorial pembangunan website **SDN Teja 1**. Kita telah melewati tahap inisialisasi, integrasi Sanity CMS, dan pengembangan antarmuka yang keren dengan gaya neo-brutalist. Sekarang, saatnya mempublikasikan karya kita ke dunia nyata.

## Persiapan Sebelum Deployment

Sebelum mengirimkan kode ke Vercel, ada beberapa hal yang harus kita pastikan:

1. **Repository GitHub:** Pastikan proyek Anda sudah di-*upload* ke GitHub (atau GitLab/Bitbucket).
2. **Environment Variables:** Website kita bergantung pada Sanity. Kita membutuhkan `NEXT_PUBLIC_SANITY_PROJECT_ID` dan `NEXT_PUBLIC_SANITY_DATASET`.
3. **Sanity CORS:** Jangan lupa untuk menambahkan URL produksi Anda (misalnya `https://sdnteja1.vercel.app`) ke dalam daftar CORS origin di dashboard Sanity agar API bisa diakses dari domain tersebut.

## Langkah 1: Hubungkan ke Vercel

Vercel adalah platform paling populer untuk men-deploy aplikasi Next.js karena integrasinya yang sangat mulus:

1. Login ke dashboard **Vercel**.
2. Klik tombol **"Add New"** lalu pilih **"Project"**.
3. Hubungkan akun GitHub Anda dan pilih repositori `sdnteja1`.

## Langkah 2: Mengatur Environment Variables

Di halaman konfigurasi project Vercel:

- Klik tab **Environment Variables**.
- Tambahkan `NEXT_PUBLIC_SANITY_PROJECT_ID` dengan value dari project Sanity Anda.
- Tambahkan `NEXT_PUBLIC_SANITY_DATASET` (biasanya `production`).
- Tambahkan API token jika Anda memerlukan akses data yang bersifat *private*.

## Langkah 3: Deploy!

Klik tombol **"Deploy"**. Vercel akan secara otomatis menjalankan proses build:

- Menginstal dependencies.
- Menjalankan `next build`.
- Mengoptimasi aset gambar dan script.

Tunggu beberapa menit, dan website Anda akan segera online dengan domain `.vercel.app`.

## Integrasi Sanity Webhook (Opsional tapi Penting)

Agar website Anda selalu menampilkan data terbaru setiap kali ada perubahan di Sanity Studio, Anda bisa menggunakan **Webhooks**. Di dashboard Sanity, tambahkan webhook yang mengarah ke URL deployment Vercel Anda untuk melakukan *revalidation* konten secara otomatis.
